# 毕业设计开题报告（与 md_phms 项目对齐稿）

> 说明：本文档依据仓库实际技术栈与功能整理，便于开题报告、论文「系统实现」章节与答辩材料一致。  
> 项目路径：`md_phms`（Maven 多模块：`phms_cpmmon`、`phms_pojo`、`phms_server`、`phms_front`）。

---

## 1.1 研究背景

随着社会经济发展与生活水平提高，公众健康意识不断增强，同时人口老龄化与慢性病年轻化趋势明显。传统医疗偏重“治病”，现代健康管理更强调日常监测与行为干预。市面健康类应用虽多，但常存在功能割裂（如仅记步或仅记饮食）、数据分散在不同设备与表格中，难以形成连续、可分析的个人健康视图，形成“数据孤岛”。因此，有必要构建一个能够**汇聚多来源健康数据、支持统一建档与趋势分析**，并在此基础上提供**可理解的评估与辅助决策**的个人健康管理系统，作为“健康伴侣”类应用的实践探索。

---

## 1.2 理论意义

本课题从工程实现角度探讨**多源异构健康数据**（如穿戴/导入类日指标、运动记录、饮食记录、用户档案与健康目标等）的**接入、规范化存储与融合展示**，并研究基于融合数据的**简单可解释健康评估**（如综合评分、风险分层、指标摘要）及**知识检索辅助问答**的可行路径。该工作可验证数据融合与健康画像构建在中小型 Web 系统中的落地方式，并为后续引入更复杂的统计模型或外部大模型接口预留接口与配置空间。

---

## 1.3 现实意义

围绕本课题实现的系统可在以下方面体现实用价值：

1. **打破来源壁垒**：通过数据源管理、文件导入与同步任务等机制，将分散的日健康指标、运动与饮食记录纳入同一用户维度，便于连续观察。
2. **可理解的健康视图**：在总览与趋势页以可视化方式呈现步数、睡眠、心率、血压等变化，降低用户理解成本。
3. **行为与内容支持**：运动课程与食谱以可检索、可评分的资源库形式支撑用户选择；健康目标与记录帮助用户跟踪阶段性指标。
4. **智能辅助扩展**：健康助手结合画像摘要与本地知识库命中内容，在默认模式下提供规则化回答，并可通过配置对接外部 AI API，兼顾可控性与扩展性。

## 2. 主要研究内容

### 2.1 多源健康数据采集与融合模块

- **数据接入**：支持用户配置多类数据源（如文件导入、手动、设备/平台等语义类型）；通过 **CSV 等文件导入**与同步任务将数据写入系统；健康指标涵盖步数、睡眠、静息心率、压力、血压等；运动与饮食分别以运动记录、饮食记录（关联食谱）形式录入。
- **数据融合与存储**：对多来源数据进行解析、去重与冲突处理，统一落入 **MySQL** 关系模型；通过数据源类型与优先级等策略支撑“同一用户、多来源”的整合视图。

### 2.2 个人健康画像与可视化模块

- **健康评估**：基于近 30 天健康指标及运动、饮食记录，采用**规则加权**生成综合评分、风险等级、亮点与改进建议，形成个人健康画像。
- **数据可视化**：前端采用 **Vue 3** 基于 **SVG** 绘制趋势图（项目内自研图表逻辑），展示健康指标及饮食/运动热量等趋势，配合总览、档案与健康目标相关页面完成多维展示。

### 2.3 智能健康干预与推荐模块（与实现粒度一致）

- **运动侧**：以**运动课程资源库**为核心，支持检索、排序与用户对课程的**评分反馈**；课程包含建议时长、单位时间耗能、难度、适用人群与频次等结构化信息，用于指导用户选择（非自动生成运动处方引擎）。
- **饮食侧**：以**食谱（推荐）库**为核心，支持用户与管理员维护食谱、图片上传与评分排序；饮食记录按食谱估算摄入热量，并与画像统计结合。
- **健康目标**：用户选择系统维护的健康目标并持续记录，支撑阶段性指标跟踪与档案内趋势展示。

### 2.4 系统架构设计与实现

- **前后端分离**：前端 `phms_front` 采用 **Vue 3 + Vite + Vue Router**；会话状态采用轻量 **reactive** 模块维护。
- **后端单体应用**：`phms_server` 为 **Spring Boot 3**（JDK 17）单体 Web 应用，通过 Maven 子模块划分公共与模型层，便于维护与扩展。
- **安全与权限**：基于 **HttpSession** 的登录态与拦截器保护 API；区分普通用户与管理员功能。
- **持久层**：**MyBatis** 访问 **MySQL**。
- **AI 与知识库**：通过 `application.yml` 中 `phms.ai`、`phms.knowledge` 配置；实现为**本地知识库检索**与可替换的**AiModelService**。

## 3.2 研究方法

1. **文献研究法**：查阅健康管理系统、多源数据整合、可解释评估、评分与推荐系统及 Web 架构等相关资料。
2. **系统设计法**：需求与用例分析、REST API 与数据库设计、关键流程（注册登录、数据源与导入、画像生成、健康助手问答、管理端维护）梳理。
3. **实验与验证法**：利用演示/种子数据与模拟导入文件，验证解析、画像计算、页面展示与管理功能；必要时对响应时间与并发做简要测试。
